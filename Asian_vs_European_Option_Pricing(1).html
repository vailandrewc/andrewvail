Asian vs European Option Pricing¶

This notebook compares the pricing behavior of Asian and European call and put options under a lognormal asset model. It includes simulation-based and analytical pricing approaches, with particular focus on the effects of path dependence.

Objective¶

To quantify the pricing impact of averaging in Asian options by comparing them with standard European options across a range of strike prices and stochastic models.

Key Concepts¶

  • European Options: Payoff depends solely on the terminal asset price at maturity. Priced using the Black-Scholes-Merton formula.
  • Asian Options (Arithmetic Average): Payoff depends on the average price over the option’s life. Priced via Monte Carlo simulation due to the lack of closed-form solution for arithmetic averaging.
  • Jump Diffusion: Simulates rare, discrete shocks in asset prices and compares path behavior with standard geometric Brownian motion.

# Jump Diffusion Simulation
lam = 0.04
mu_J = -0.005
sigma_J = 0.10

S0, sigma, r, T = 100, 0.20, 0.02, 1
sim_sigma, sim_r = sigma / np.sqrt(365), r / 365,
sim_days = int(np.floor(365*T))

S = np.zeros(sim_days)
S[0] = S0
Z = np.random.normal(0, 1, sim_days)

jump_times = []

for t in range(1, sim_days):
    if np.random.poisson(lam=lam):
        J = np.random.lognormal(mu_J + sigma_J**2/2, sigma_J) - 1
        jump_times.append(t)
    else:
        J = 0

    S[t] = S[t-1] * (1 + sim_r + sim_sigma*Z[t] + J)

fig, ax = plt.subplots(figsize=(9, 5))
ax.plot(S, label="Price Path")
ax.set_xlim(-20, 370)

ymin, ymax = ax.get_ylim()
ax.vlines(x=jump_times, ymin=ymin, ymax=ymax, color="red", linewidth=0.5, label="Jump Events")

ax.set_title("Simulated Jump Diffusion Path")
ax.set_xlabel("Days")
ax.set_ylabel("Price")
ax.legend()
plt.grid(True)
plt.show()

def euro_option_prices_jump_diffusion(
    S0, K, T, r, sigma,
    lam, mu_J, sigma_J,
    runs=10000, steps=365
):
    """
    Monte Carlo pricing of European call and put options
    under Merton's Jump-Diffusion model.
    """
    dt = T / steps

    # Risk-neutral drift adjusted for jump intensity
    kappa = np.exp(mu_J + 0.5 * sigma_J**2) - 1
    drift = (r - lam * kappa - 0.5 * sigma**2) * dt
    diffusion = sigma * np.sqrt(dt)

    S_paths = np.zeros((runs, steps))
    S_paths[:, 0] = S0

    for t in range(1, steps):
        Z = np.random.randn(runs)
        jump_occurs = np.random.rand(runs) < lam * dt

        log_J = np.random.normal(mu_J, sigma_J, size=runs)
        jump_multiplier = np.ones(runs)
        jump_multiplier[jump_occurs] = np.exp(log_J[jump_occurs])

        S_paths[:, t] = S_paths[:, t-1] * np.exp(drift + diffusion * Z) * jump_multiplier

    S_T = S_paths[:, -1]
    call_payoff = np.maximum(S_T - K, 0)
    put_payoff = np.maximum(K - S_T, 0)

    call_price = np.exp(-r * T) * np.mean(call_payoff)
    put_price = np.exp(-r * T) * np.mean(put_payoff)

    return call_price, put_price

def simulate_asian_call_put(S0, sigma, r, T, runs, K, steps=365):
    dt = T / steps
    drift = (r - 0.5 * sigma**2) * dt
    diffusion = sigma * np.sqrt(dt)

    call_prices = []
    put_prices = []

    for _ in range(runs):
        Z = np.random.randn(steps)
        S = np.zeros(steps)
        S[0] = S0

        for t in range(1, steps):
            S[t] = S[t-1] * np.exp(drift + diffusion * Z[t])

        avg_price = np.mean(S[1:])  # exclude initial day

        call_payoff = max(0, avg_price - K)
        put_payoff = max(0, K - avg_price)

        call_prices.append(np.exp(-r * T) * call_payoff)
        put_prices.append(np.exp(-r * T) * put_payoff)

    return np.mean(call_prices), np.mean(put_prices)

from scipy.stats import norm

def bs_call(S, K, T, r, sigma):
    d1 = (np.log(S / K) + (r + 0.5 * sigma**2) * T) / (sigma * np.sqrt(T))
    d2 = d1 - sigma * np.sqrt(T)
    return S * norm.cdf(d1) - K * np.exp(-r * T) * norm.cdf(d2)

def bs_put(S, K, T, r, sigma):
    d1 = (np.log(S / K) + (r + 0.5 * sigma**2) * T) / (sigma * np.sqrt(T))
    d2 = d1 - sigma * np.sqrt(T)
    return K * np.exp(-r * T) * norm.cdf(-d2) - S * norm.cdf(-d1)

Asian options are less sensitive to spikes in the underlying asset’s price because they average over time. This averaging reduces effective volatility, lowering the value of both calls and puts compared to their European counterparts. The difference is most significant when the option is deep in- or out-of-the-money.
